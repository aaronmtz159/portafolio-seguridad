Mecanismos de Defensa en Red

Configuración de Políticas de Seguridad Mediante iptables

Estudiante: Aaron Efraim Mata Martinez (179419)
Materia: Seguridad Informática | Fecha: 03/02/2026

1. Introducción Contextual

La seguridad perimetral es la primera línea de defensa de cualquier infraestructura digital. El uso de herramientas como iptables permite administrar el flujo de paquetes en sistemas Linux, aplicando políticas que garantizan la integridad de la red. En esta actividad, se exploran las configuraciones de filtrado para servicios críticos (DNS, HTTP, Mail) bajo una postura de seguridad restrictiva.

2. Desarrollo Técnico

Postura de Seguridad: Denegar por Defecto

Se establece una política restrictiva global para descartar cualquier tráfico de entrada no autorizado.

iptables -P INPUT DROP
Nota: Esta acción asegura que solo el tráfico explícitamente definido pueda ingresar al sistema.

Reglas de Control de Flujo (FORWARD)

Implementación de inspección de estado para permitir tráfico legítimo:

iptables -A FORWARD -m state --state ESTABLISHED, RELATED -j ACCEPT
iptables -A FORWARD -p tcp -s 192.1.2.0/24 -d 0.0.0.0/0 --dport 53 -m state --state NEW -j ACCEPT

Gestión de Servicios de Red

Configuración para servidores de correo y servidores web corporativos:

! Correo Entrante y Saliente
iptables -A FORWARD -p tcp -s 0.0.0.0/0 -d 192.1.2.10 --dport 25 -m state --state NEW -j ACCEPT
iptables -A FORWARD -p tcp -s 192.1.2.10 -d 0.0.0.0/0 --dport 25 -m state --state NEW -j ACCEPT

! Servidor Web Local
iptables -A FORWARD -p tcp -s 0.0.0.0/0 -d 192.1.2.11 --dport 80 -m state --state NEW -j ACCEPT

4. Reflexión Técnica

Implementar mecanismos de defensa basados en Stateful Inspection es vital para prevenir ataques de suplantación y asegurar que solo las conexiones solicitadas por el cliente sean procesadas. La transición de una política permisiva a una restrictiva (DROP) eleva significativamente el nivel de seguridad, obligando al administrador a conocer profundamente los puertos y protocolos necesarios para la operatividad del negocio.
