1. Fundamentos del Flujo de Paquetes

El procesamiento de un paquete en iptables sigue una jerarquía lógica: primero atraviesa una tabla, luego se clasifica en una cadena y finalmente se aplica una acción específica.

Propósito de las Tablas

Tabla Propósito Principal Ejemplo de Uso
FILTER Filtrado de paquetes para permitir o bloquear tráfico. Reglas básicas de Firewall.
NAT Traducción de direcciones de red. Reenvío de puertos (Port Forwarding).
MANGLE Modificación técnica de los paquetes. Calidad de Servicio (QoS).
RAW Configuración de excepciones de seguimiento. Evitar Notrack.
SECURITY Integración con políticas de SELinux. Etiquetas de seguridad avanzada.

2. Anatomía y Traducción de Comandos

Comando de ejemplo:

iptables -A INPUT -p tcp -multiport --dports 80,443 -j ACCEPT
Traducción: Esta regla permite el tráfico de entrada para los protocolos HTTP y HTTPS a través de los puertos 80 y 443 respectivamente.

3. Casos de Aplicación Técnica

Configuración de políticas por defecto:

iptables -P OUTPUT ACCEPT
Establece que todo el tráfico que sale del equipo sea permitido de forma predeterminada.

Acceso SSH Restringido:

iptables -A INPUT -p tcp -s 192.168.1.50 --dport 22 -j ACCEPT
Permite el acceso vía SSH (puerto 22) exclusivamente desde la dirección IP 192.168.1.50.

Seguridad de Estado (Stateful Inspection):

iptables -A INPUT -i eth0 -p tcp -m multiport --dports 22,80,443 -m state --state NEW,ESTABLISHED -j ACCEPT
Esta regla avanzada en la tabla filter permite conexiones SSH y Web por la interfaz eth0, pero solo si son nuevas solicitudes o conexiones ya establecidas.

4. Variables y Flags de Administración

  • Control de Tasa: --limit 5/minute (limita intentos por tiempo).
  • Filtrado de Red: -s 192.168.1.0/24 (aplica a un segmento de red).
  • Visualización Técnica: -L -n -v (lista reglas con contadores y sin resolución DNS).
